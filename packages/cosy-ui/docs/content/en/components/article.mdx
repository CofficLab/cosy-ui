# Article

The Article component is used to display formatted article content with a complete set of typography styles, making content more readable and understandable. This component is suitable for blog posts, documentation pages, news content, and other scenarios requiring good typography.

## Basic Usage

Wrap your content with the Article component:

<CodeExample
  code={`<Article>
  <h1>Article Title</h1>
  <p>This is article content that can include <a href="#">links</a> and <code>inline code</code>.</p>
  
  <h2>Second Level Heading</h2>
  <p>This is a paragraph under the second level heading.</p>
</Article>`}
>
  <Article>
    <h1>Article Title</h1>
    <p>This is article content that can include <a href="#">links</a> and <code>inline code</code>.</p>
    
    <h2>Second Level Heading</h2>
    <p>This is a paragraph under the second level heading.</p>
  </Article>
</CodeExample>

## Supported Elements

The Article component automatically applies styles to the following elements:

- Headings (h1-h6)
- Paragraphs (p)
- Lists (ul, ol, li)
- Links (a)
- Blockquotes (blockquote)
- Code (code, pre)
- Images (img)
- Horizontal rules (hr)
- Tables (table, th, td)

## Properties

| Property | Type | Default | Description |
| ------ | ---- | ------ | ---- |
| class | string | - | Custom class name |
| width | string | "medium" | Article width, supports narrow, medium, wide, or full |
| style | string | - | Inline style |

## Best Practices

- Use appropriate HTML semantic tags to maintain structured content
- Choose narrower widths for long article content to improve readability
- Use the Article component with other components (like TableOfContents) to provide better content navigation
- In responsive layouts, consider using narrower widths on mobile devices
