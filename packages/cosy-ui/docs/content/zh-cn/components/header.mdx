Header 组件是一个用于网站顶部导航的布局组件，提供了logo、导航菜单、语言切换等功能。组件支持响应式设计，在不同屏幕尺寸下有良好的显示效果，并可选择固定在页面顶部。

## 基础用法

基本的 Header 组件包含 logo 和语言切换功能：

```astro
---
import { Header } from 'cosy-ui';
---

<Header
  logo={logoImage}
  logoHref="/"
/>
```

## 带导航菜单

通过 `navItems` 属性添加导航菜单项：

```astro
---
import { Header } from 'cosy-ui';
---

<Header
  logoHref="/"
  navItems={[
    { 
      href: "/docs", 
      label: "文档", 
      match: (path) => path.startsWith("/docs") 
    },
    { 
      href: "/components", 
      label: "组件", 
      match: (path) => path.startsWith("/components") 
    },
    { 
      href: "/blog", 
      label: "博客", 
      match: (path) => path.startsWith("/blog") 
    }
  ]}
/>
```

`match` 函数用于确定当前页面是否与导航项匹配，如果匹配则应用活动样式。

## 自定义高度

使用 `height` 属性可以设置导航栏的高度，支持七种预设大小：

```astro
---
import { Header } from 'cosy-ui';
import logoImage from '../assets/logo.png';
---

<Header
  logo={logoImage}
  height="sm" // 可选值: "3xs", "2xs", "xs", "sm", "md", "lg", "xl"
/>
```

不同高度对比：

- `3xs`: 微小尺寸 (16px) - 适合极简界面或内嵌组件
- `2xs`: 超小尺寸 (24px) - 适合紧凑型工具栏
- `xs`: 特小尺寸 (32px) - 适合辅助导航栏
- `sm`: 小尺寸 (40px) - 适合轻量级工具类网站
- `md`: 中等尺寸 (48px，默认) - 适合大多数应用
- `lg`: 大尺寸 (64px) - 适合内容中心的网站
- `xl`: 超大尺寸 (80px) - 适合品牌展示型网站

高度设置会自动调整 logo 大小和页面顶部的占位空间（当 `sticky` 为 true 时）。

## 自定义语言选项

自定义语言切换下拉菜单中的选项：

```astro
---
import { Header } from 'cosy-ui';
import logoImage from '../assets/logo.png';
---

<Header
  logo={logoImage}
  languages={[
    { code: "zh-cn", name: "简体中文" },
    { code: "en", name: "English" },
    { code: "ja", name: "日本語" }
  ]}
  currentLocale="zh-cn"
/>
```

## 固定在顶部

使用 `sticky` 属性可以使导航栏固定在页面顶部：

```astro
---
import { Header } from 'cosy-ui';
import logoImage from '../assets/logo.png';
---

<Header
  logo={logoImage}
  sticky={true}
/>
```

## 带基础路径

对于部署在子目录下的网站，可以使用 `basePath` 属性处理路径：

```astro
---
import { Header } from 'cosy-ui';
import logoImage from '../assets/logo.png';
---

<Header
  logo={logoImage}
  basePath="/my-site"
/>
```

## 属性

| 属性名 | 类型 | 默认值 | 说明 |
| ------ | ---- | ------ | ---- |
| logo | ImageMetadata | - | 必填，网站 logo 图片 |
| logoHref | string | "/" | Logo 链接地址 |
| navItems | array | [] | 导航项数组，每项包含 href、label 和 match 函数 |
| languages | array | [中文, English] | 语言选项数组，每项包含 code 和 name |
| currentLocale | string | "zh-cn" | 当前语言代码 |
| sticky | boolean | true | 是否固定在页面顶部 |
| basePath | string | "" | 网站基础路径，用于部署在子目录的情况 |
| height | string | "md" | 导航栏高度: "sm", "md", "lg", "xl" |
| socialLinks | array | - | 社交媒体链接数组 |
| showSidebarToggle | boolean | false | 是否显示侧边栏切换按钮 |
| defaultSidebarOpen | boolean | false | 侧边栏是否默认展开 |

**NavItem 类型：**
```typescript
interface NavItem {
  href: string;       // 导航链接
  label: string;      // 导航标签文本
  match: (path: string) => boolean;  // 判断是否为当前活动项的函数
}
```

**Language 类型：**
```typescript
interface Language {
  code: string;   // 语言代码
  name: string;   // 语言显示名称
}
```

## 最佳实践

- 为了良好的用户体验，建议在移动设备上使用固定的页头，并搭配汉堡菜单
- 导航项数量不宜过多，通常4-6个为宜，过多时考虑使用下拉菜单分组
- 为多语言网站设置清晰的语言切换按钮，并确保当前语言明显标识
- 使用 `match` 函数提供精确的当前路径匹配逻辑，为用户提供明确的位置指示
- 当使用 `sticky` 属性时，注意设置适当的内容边距，避免内容被固定的页头遮挡
- 选择适合您网站风格的高度，一般而言，营销网站可以使用较大高度，而工具类网站通常选择较小高度以节省垂直空间
