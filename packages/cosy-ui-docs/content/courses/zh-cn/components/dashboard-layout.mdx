## 简介

DashboardLayout 组件适用于管理后台的布局，包含侧边栏导航和顶部导航栏。提供了完整的管理界面框架，包括响应式设计和暗色主题支持。

组件特性：

- 🏗️ **完整框架** - 提供侧边栏、顶部导航栏和主内容区的完整布局
- 📱 **响应式设计** - 在不同设备上提供良好的用户体验
- 🎨 **daisyUI样式** - 使用daisyUI组件库提供一致的设计风格
- 🔧 **高度可定制** - 支持自定义导航项、系统名称和用户信息
- 💾 **状态保持** - 记住用户的侧边栏折叠状态
- 🌙 **主题支持** - 完整支持明暗主题切换

## 基础用法

最简单的仪表盘布局使用方式：

<DashboardLayoutPackage.DashboardLayoutContainers.Basic />

## 自定义样式

通过class属性自定义仪表盘的外观样式：

## 使用示例

### 基本配置

```astro
---
import { DashboardLayout } from '@coffic/cosy-ui';

// 定义导航项
const navItems = [
  { href: "/dashboard", icon: "home", text: "仪表盘" },
  { href: "/dashboard/users", icon: "users", text: "用户管理", badge: "5" },
  { 
    href: "/dashboard/content", 
    icon: "document", 
    text: "内容管理",
    items: [
      { href: "/dashboard/content/articles", icon: "document", text: "文章管理" },
      { href: "/dashboard/content/categories", icon: "star", text: "分类管理" }
    ]
  },
  { href: "/dashboard/settings", icon: "settings", text: "系统设置" }
];
---

<DashboardLayout
  title="管理后台"
  navItems={navItems}
  userName="管理员"
  systemName="内容管理系统"
>
  <h1>欢迎使用管理后台</h1>
  <p>这里是主要内容区域</p>
</DashboardLayout>
```

## Props 参考

### DashboardLayout Props

| 属性               | 类型                 | 必需 | 默认值     | 描述                     |
| ------------------ | -------------------- | ---- | ---------- | ------------------------ |
| `title`            | `string`             | ✅   | -          | 页面标题                 |
| `description`      | `string`             | ❌   | -          | 页面描述                 |
| `systemName`       | `string`             | ❌   | "管理系统" | 系统名称，显示在侧边栏   |
| `navItems`         | `NavItem[]`          | ✅   | -          | 导航项目数组             |
| `userName`         | `string`             | ❌   | -          | 用户名，显示在顶部导航栏 |
| `userAvatar`       | `string`             | ❌   | -          | 用户头像URL              |
| `sidebarCollapsed` | `boolean`            | ❌   | `false`    | 是否折叠侧边栏           |
| `sidebarSize`      | `'sm' \| 'md' \| 'lg' \| 'xl'` | ❌   | `'md'`     | 侧边栏尺寸大小           |
| `sidebarTheme`     | `SidebarTheme`       | ❌   | `'default'`| 侧边栏主题色彩           |
| `contentTheme`     | `ContentTheme`       | ❌   | `'card'`   | 内容区域主题样式         |
| `head`             | `astroHTML.JSX.Element` | ❌   | -          | 自定义头部内容           |
| `class`            | `string`             | ❌   | -          | 页面类名                 |
| `class:list`       | `any`                | ❌   | -          | 类名列表                 |

interface NavItem {

## 功能特性

### 响应式设计

- **桌面端**：显示完整的侧边栏和所有功能
- **平板端**：侧边栏可折叠，保持良好的交互体验
- **移动端**：侧边栏转为抽屉模式，节省屏幕空间

### 侧边栏尺寸配置

- **SM (192px)**：紧凑布局，移动端友好，适合简单导航
- **MD (256px)**：默认尺寸，平衡的选择，适合大多数场景
- **LG (288px)**：宽松布局，适合更多内容的展示
- **XL (320px)**：最宽布局，适合复杂的多级导航结构

### 侧边栏主题系统

DashboardLayout 提供10种预设主题，基于 daisyUI 的语义化颜色系统：

- **default**：默认浅灰色，通用场景的最佳选择
- **dark**：深色主题，现代感设计，减少视觉疲劳
- **neutral**：中性色调，专业商务场景
- **primary**：品牌主色，强调企业特色
- **secondary**：辅助色调，温和的视觉效果
- **accent**：强调色，用于重点功能突出
- **info**：信息蓝色，适合数据展示界面
- **success**：成功绿色，健康、财务等正面应用
- **warning**：警告黄色，监控、预警类系统
- **error**：错误红色，安全、审计等严肃应用

所有主题都自动处理文字对比度，确保良好的可读性和无障碍访问。

### 内容主题系统

DashboardLayout 提供11种内容区域主题，可以灵活配置主要内容的背景样式：

- **card**：默认卡片样式，提供清晰的边界和轻微阴影效果
- **transparent**：透明背景，移除容器限制，提供最大的设计自由度
- **base**：基础背景色，使用 base-200 色调，温和的视觉效果
- **neutral**：中性色调，专业商务感
- **primary**：品牌主色，强化企业识别度
- **secondary**：辅助色调，温和的品牌体现
- **accent**：强调色，用于重点内容突出
- **info**：信息蓝色，适合数据展示和分析界面
- **success**：成功绿色，健康管理、财务增长等正面应用
- **warning**：警告黄色，监控系统、待处理状态等需要注意的场景
- **error**：错误红色，安全审计、错误处理等严肃应用

内容主题系统特别适合：
- **管理后台**：card 主题提供专业、清晰的管理界面
- **创意工具**：transparent 主题提供无限制的设计空间
- **品牌展示**：primary/secondary/accent 主题强化品牌特色
- **数据看板**：info 主题营造专业的数据分析氛围
- **状态监控**：warning/error 主题突出系统状态和问题

### 状态管理

- 自动保存侧边栏折叠状态到本地存储
- 页面刷新后恢复用户的使用习惯
- 支持编程控制侧边栏状态

### 导航系统

- 支持多级导航结构
- 自动高亮当前页面对应的导航项
- 支持徽章显示（如消息数量、待处理任务等）

## 最佳实践

### 导航结构设计

1. **保持简洁** - 主导航项建议不超过7个
2. **逻辑分组** - 相关功能归类到子菜单中
3. **重要性排序** - 常用功能放在顶部
4. **图标一致** - 使用语义化的图标提升识别度

### 内容区域布局

1. **卡片包装** - 使用 `card` 组件包装内容区域
2. **间距统一** - 使用一致的内外边距
3. **响应式考虑** - 确保内容在各种屏幕尺寸下的可读性

### 性能优化

1. **按需加载** - 大型页面考虑懒加载
2. **状态缓存** - 合理利用本地存储
3. **组件复用** - 提取公共组件避免重复

## 样式定制

DashboardLayout 组件遵循 cosy-ui 的样式规范：

- 使用 daisyUI 的 `drawer`、`navbar`、`menu`、`card` 等组件
- Tailwind CSS 类需要添加 `cosy:` 前缀（如 `cosy:bg-base-200`）
- 支持完整的响应式设计
- 内置明暗主题支持

```css
/* 自定义样式示例 */
.custom-dashboard {
  background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
}
```
